# BMS Proxy Service

A NestJS backend service that acts as a proxy for your frontend to communicate with an external BMS API.

## Features

- **Authentication Proxy**: Handles BMS login and manages session cookies
- **Request Forwarding**: Forwards requests to BMS API with proper authentication
- **Cookie Management**: Automatically stores and manages JSESSIONID and DWRSESSIONID cookies
- **Error Handling**: Comprehensive error handling for expired sessions and failed requests

## Installation

```bash
npm install
```

## Running the Service

```bash
# Development mode
npm run start:dev

# Production mode
npm run start:prod
```

The service will run on port 4000.

## API Endpoints

### 1. Login Endpoint

**POST** `/api/login`

Authenticates with the BMS API and stores session cookies.

**Request Body:**
```json
{
  "username": "crossbow",
  "password": "crossbow@123"
}
```

**Response:**
```json
{
  "message": "Login successful",
  "cookies": {
    "JSESSIONID": "abc123...",
    "DWRSESSIONID": "def456..."
  }
}
```

### 2. Set Temperature Endpoint

**POST** `/api/set-temp`

Forwards temperature configuration requests to the BMS API.

**Request Body:**
```json
{
  "device_vfdReadingSetTemp": 26,
  "requestType": "ElectricalMeters",
  "subRequestType": "ConfigureDataAtDevice",
  "username": "crossbow"
}
```

**Response:**
Returns the BMS API response directly.

## Usage Flow

1. **Login First**: Call `/api/login` with your credentials
2. **Make Requests**: Use `/api/set-temp` for temperature configuration
3. **Session Management**: Cookies are automatically managed and included in requests

## Error Handling

- **No Active Session**: Returns 400 if no valid session exists
- **Session Expired**: Automatically clears cookies and returns 400
- **Login Failed**: Returns error message if BMS authentication fails

## Architecture

- **AuthService**: Handles BMS login and cookie storage
- **ProxyService**: Forwards requests with proper authentication
- **Controllers**: Expose REST endpoints for frontend communication
- **Modules**: Proper dependency injection and module organization

## Dependencies

- `@nestjs/common`: NestJS core functionality
- `axios`: HTTP client for external API calls
- `querystring`: URL encoding for form data
